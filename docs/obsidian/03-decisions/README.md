# Architecture Decision Records (Vault Slice)

Historical ADRs live in [`docs/06-DECISIONS.md`](../../06-DECISIONS.md) as a single file. **New ADRs** go here as individual files from this point forward. Eventually the old single-file ADRs may be split out.

## Format

Files are named `YYYY-MM-DD-<slug>.md`. Each ADR follows this shape:

```markdown
---
title: "Descriptive title"
date: YYYY-MM-DD
status: proposed | accepted | superseded-by-<slug> | deprecated
supersedes: (if any)
last_verified_sha: <HEAD at decision time>
tags: [decision, adr]
---

## Context
## Decision
## Alternatives Considered
## Consequences
## Review Date
```
