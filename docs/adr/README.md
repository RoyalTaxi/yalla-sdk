# Architecture Decision Records

Short, durable records of decisions that shape the SDK — one file per decision, in
`NNNN-title.md`, in **decision / context / consequences** form. An ADR captures *why*
a choice was made so the next reader does not relitigate it from scratch.

Status values: `proposed` · `accepted` · `superseded by NNNN`.
